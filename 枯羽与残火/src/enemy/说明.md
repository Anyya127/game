# Enemy 模块 — 代码架构说明

## 文件

- `enemy_api.h` — 对外接口
- `enemy.c` — 实现

## 数据结构

静态数组池 `EnemyData g_enemies[MAX_ENEMIES]`（20个槽位），用 `active` 标志位管理槽位复用。创建时找空槽位填入，删除时置 `active = false`。不移动内存，ID 保持不变。

```c
typedef struct {
    Vec2 pos, vel;
    AIState state;          // 当前状态机状态
    EnemyType type;         // BASIC / ARMORED / BOSS
    float detectRange;      // 发现玩家的距离阈值
    float attackCooldown;   // 攻击间隔，默认2秒
    bool wantsShoot;        // 本帧想射子弹（AI决策设true，main.c消费）
    ...
} EnemyData;
```

## 核心流程：Enemy_UpdateAll()

遍历所有活跃敌人，每个执行两件事：

### 1. AI决策（updateAI）

状态机转换图：
```
IDLE ──(2秒后)──→ PATROL ──(发现玩家)──→ CHASE ──(够近)──→ ATTACK
                    ↑                        ↑                   │
                    └──(追丢了)──────────────┘                   │
                                                                  │
                    └──────(玩家跑远)─────────────────────────────┘
```

各状态行为：

- **IDLE**：站2秒不动，然后转 PATROL
- **PATROL**：慢速左右走（速度2px），走1秒停1秒后转身。检测到玩家进 `detectRange`→ 转 CHASE
- **CHASE**：高速追玩家（速度8px）。距离 < `detectRange * 0.7` → 转 ATTACK。距离 > `detectRange * 1.5` → 转 PATROL
- **ATTACK**：站桩不动，每 `attackCooldown` 秒设 `wantsShoot = true`。玩家跑远 → CHASE
- **AI_DYING / AI_DEAD**：不动

三种敌人的差异化参数（在 `Enemy_Create` 的 switch 中设置）：

| 参数 | BASIC | ARMORED | BOSS |
|------|-------|---------|------|
| HP | 50 | 100 | 500 |
| 体积 | 40×60 | 50×70 | 100×120 |
| 视野 | 300px | 250px | 400px |
| 接触伤害 | 10 | 20 | 30 |

### 2. 物理（applyPhysics）

跟玩家一样：重力 + 摩擦力 + 速度移动。目前只做地面站立，未做平台碰撞。

## 战斗接口

`Enemy_TakeDamage()` 扣血 → HP到0 → 标记 `AI_DEAD` + `active = false` + `g_count--`。

`Enemy_GetAll()` 拷贝所有活跃敌人数据到外部缓冲。拷贝不暴露内部指针，外部修改不影响内部状态。

## 与其他模块的关系

| 谁调我 | 调什么 |
|--------|--------|
| main.c | `Enemy_UpdateAll()`, `Enemy_GetAll()`, `Enemy_Spawn()` |
| combat.c | `Enemy_GetAll()` (遍历做碰撞) |
| level.c | 不调我，只产出 `SpawnRequest` 给 main.c |
| render.c | `Enemy_GetAll()` (拿数据绘图) |
