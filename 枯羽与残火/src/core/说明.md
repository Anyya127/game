# Core 模块 + main.c — 代码架构说明

## 文件

- `core_api.h` / `core.c` — 底层封装
- `main.c` — 主循环装配

## Core 模块职责

封装 raylib，让其他模块不直接依赖 raylib。

### 窗口
`Core_InitWindow(w, h, title)` → `InitWindow` + `SetTargetFPS(60)`。

`Core_ShouldClose()` → `WindowShouldClose()`。

### 帧控制
`Core_DeltaTime()` → `GetFrameTime()`（60fps时约0.0167秒）。

`Core_BeginFrame()` → `BeginDrawing()` + `ClearBackground(WHITE)`。

`Core_EndFrame()` → `EndDrawing()`。

### 输入采集：Core_PollInput()

把键盘和鼠标映射成 `PlayerInput` 结构体。多键位方案：
- 移动：方向键 / WASD（`IsKeyDown`，允许长按）
- 跳跃：空格 / W / 上方向（`IsKeyDown`）
- 射击：Z / J / 鼠标左键（`IsKeyDown`）
- 弹反：X / 鼠标右键
- 暂停：P / ESC（`IsKeyPressed`，只触发一次）
- 重启：R

`IsKeyDown` vs `IsKeyPressed` 区别：`IsKeyDown` 每帧只要按住就返回true，`IsKeyPressed` 只在按下的第一帧true。

### 数学工具箱

- `Util_Clamp(v, min, max)`：限制值范围
- `Util_Lerp(a, b, t)`：`a + (b-a)*t`，线性插值
- `Util_Distance(x1,y1, x2,y2)`：两点距离
- `Util_Random(min, max)`：`rand() % (max-min+1) + min`

## main.c：主循环装配

main.c 是"胶水层"——不写具体业务逻辑，只按顺序调用各模块的 API。

### 初始化顺序（main函数开头）

```
Core_InitWindow → Render_Init → Player_Init → Enemy_Init → Combat_Init → Level_Init → Audio_PlayBGM
```

### 每帧逻辑顺序（暂停时跳过逻辑部分）

```
 1. Level_Update()           → 摄像机 + 敌人生成计时
 2. Level_GetPlatforms()     → 拿平台列表
 3. Player_Update()          → 输入→移动→碰撞，返回 pd
 4. if pd.wantsShoot → Bullet_Create()  → 玩家子弹
 5. processSpawns()          → 消费敌人生成队列
 6. Enemy_UpdateAll()        → 全部敌人AI + 物理
 7. 遍历敌人wantsShoot → Bullet_Create() → 敌人子弹
 8. Bullet_UpdateAll()       → 移动所有子弹
 9. Combat_Update()          → 碰撞检测 → 写事件队列
10. processCombatEvents()    → 消费事件：扣血/加分/死亡
```

顺序很重要：先移动（1-8），再碰撞（9），再结算（10）。保证渲染时看到的画面和碰撞结果一致。

### 每帧渲染顺序

```
Core_BeginFrame
  → Render_GameBackground    (最远层，视差)
  → Render_Platform[]        (地形)
  → Render_Enemy[]           (敌人)
  → Render_Bullet[]          (子弹)
  → Render_Player            (玩家，画在最上面)
  → Render_UI                (HUD，最顶层)
  → if paused → Render_Pause (遮罩盖在一切上面)
Core_EndFrame
```

### 暂停机制

暂停时调用 `Level_Pause()` → `g_paused = true` → 逻辑更新跳过 → 但渲染仍然执行（画面冻结但能看到暂停界面）。

暂停检测用 `IsKeyPressed`（单帧触发），防止来回闪烁。

### 事件消费：processCombatEvents()

combat 产出的 `Event` 队列有两个消费者：
1. **VFX_ProcessEvents**（先消费）→ 震屏/hitstop 特效
2. **这里**（后消费）→ 扣血/加分/死亡判定

两者都读的是同一份事件（`Combat_PollEvents` 拷贝出来），互不干扰。消费完 `Combat_ClearEvents` 清空队列。

## 与其他模块的关系

Core 是纯被调用方，main.c 是纯调用方。main.c 只依赖 `*_api.h`，不看任何 `.c` 实现。
