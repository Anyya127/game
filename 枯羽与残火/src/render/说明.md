# Render 模块 — 代码架构说明

## 文件

- `render_api.h` — 对外接口
- `render.c` — 实现

## 职责

三块：**纹理管理**、**精灵绘制**、**音频播放**。是唯一直接调 raylib 绘图函数的模块。还消费 combat 事件做震屏/hitstop。

## 数据结构

```c
Texture2D g_tex[TEX_COUNT];     // 所有纹理的数组，用 TexId 枚举索引
Music g_bgm[BGM_COUNT];         // 背景音乐流
char g_resPath[256];            // resources 目录的绝对路径
```

`TexId` 枚举把所有纹理统一管理：
```c
typedef enum {
    TEX_BG_MENU, TEX_BG_GAME,               // 背景
    TEX_PLAYER_IDLE, TEX_PLAYER_RUN, ...     // 玩家精灵
    TEX_ENEMY_BASIC, TEX_ENEMY_ARMORED, ...  // 敌人精灵
    TEX_BULLET_PLAYER, TEX_BULLET_ENEMY,     // 子弹精灵
    TEX_TILE,                                 // 地图瓦片
    TEX_COUNT  // 总数
} TexId;
```

## 资源加载流程

### 初始化

```
Render_Init(NULL)
  ├─ findResourcePath() → 搜索 resources/ 目录 (按顺序尝试4个候选路径)
  ├─ loadAllTextures() → 遍历 TEX_COUNT 加载每个 .png
  └─ audioInit()       → InitAudioDevice + 加载BGM流
```

### 资源路径搜索策略

依次尝试：
1. `<执行文件路径>/resources`
2. `<执行文件路径>/../resources`（CMake 在 build/ 下）
3. `<执行文件路径>/../../resources`（MSVC 在 x64/Debug/ 下）
4. `<执行文件路径>/resources`（直接拼接）

用 `stat()` 检查目录是否存在，找到第一个就停。

## 绘制流程

所有精灵最终调用 `drawSprite()`：
```c
static void drawSprite(Texture2D tex, float x, float y, float w, float h,
                        bool flip, Color tint)
```

**翻转原理**：设 `source.width = -tex.height`（负值），`DrawTexturePro` 自动水平翻转。

**世界坐标 → 屏幕坐标**：`screenX = worldX - scrollX`。所有绘制都减去摄像机偏移。

### 各绘制函数的细节

- **Render_Player**：根据状态选贴图，无敌帧奇数帧不画（闪烁效果）。血条在头顶
- **Render_Enemy**：视锥裁剪（`sx + w < 0 || sx > SCREEN_W` 跳过）。装甲/Boss 画血条，杂兵不画
- **Render_Bullet**：玩家/敌人选不同贴图
- **Render_Platform**：瓦片平铺。没有贴图时 fallback 纯色方块
- **Render_GameBackground**：背景按 `cameraX * 0.3` 速度滚动（视差效果）

## 音频

只有背景音乐，用 raylib 的 `Music` 流式播放（大文件不占内存）。

`Render_UpdateAudio()` 必须每帧调用一次，驱动流缓冲。

`Audio_PlayBGM(type, volume)` 先停掉正在播的，再播新的。

## 特效

`VFX_ProcessEvents()` 遍历 combat 事件 → 不同事件类型触发不同震屏参数：
- EVT_HIT：轻震 (3px, 0.1s)
- EVT_PARRY：重震 (8px, 0.2s) + hitstop 4帧
- EVT_KILL：中震 (5px, 0.15s) + hitstop 6帧
- EVT_PLAYER_HURT：轻震 (4px, 0.15s)
- EVT_PLAYER_DEAD：巨震 (10px, 0.5s)
