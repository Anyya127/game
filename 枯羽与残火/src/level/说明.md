# Level 模块 — 代码架构说明

## 文件

- `level_api.h` — 对外接口
- `level.c` — 实现

## 职责

三件事：
1. **平台生成**：关卡加载时一次性创建所有平台
2. **摄像机**：跟随玩家（平滑插值）+ 震屏抖动
3. **敌人生成队列**：定时产生 `SpawnRequest`，由 main.c 消费

还管理游戏流程状态机。

## 数据结构

```c
PlatformData g_platforms[MAX_PLATFORMS];  // 50个平台槽位
float g_cameraX, g_scrollX;               // 摄像机位置（平滑/实际）
float g_shakeIntensity, g_shakeTimer;     // 震屏参数
GameState g_state;                        // 菜单/游戏中/暂停/结束
SpawnRequest g_spawnQueue[8];             // 本帧待生成的敌人
int g_spawnQueueLen;                      // 队列长度
```

## 核心流程

### 关卡加载：Level_Load()

顺序生成三种平台：
1. **底面**：`(0, SCREEN_H-10, MAP_TOTAL, 20)` — 覆盖全地图宽度
2. **阶梯平台**：每隔300px一个，高度循环 `SCREEN_H - 100 - (i%3)*80`，宽120~200随机
3. **高平台**：隔一个阶梯平台加一个，高度比相邻阶梯平台高100px

所有平台 `solid = true, active = true`。

### 每帧更新：Level_Update()

三个子步骤：

#### 1. 摄像机跟随
```c
target = playerX - SCREEN_W / 2          // 玩家在屏幕中央
target = clamp(target, 0, MAP_TOTAL - SCREEN_W)  // 不超出地图边界
g_cameraX = lerp(g_cameraX, target, 0.15) // 平滑插值（惯性感）
g_scrollX = g_cameraX
```
插值系数 0.15：每帧移动当前差距的15%，越靠近越慢——营造摄像机"有惯性"的感觉。

#### 2. 震屏衰减
```c
if (g_shakeTimer > 0) {
    g_shakeTimer -= dt;
    if (g_shakeTimer <= 0) g_shakeIntensity = 0;
}
```
`Level_GetScrollX()` 返回 `g_scrollX + random(-intensity, intensity)`，震屏效果自然叠加在摄像机偏移上。

#### 3. 敌人生成
每2秒生成一批（前后各一个），写入 `g_spawnQueue[]`：
- 70% 杂兵、25% 装甲兵、5% Boss
- 前方生成距玩家 `SCREEN_W + 100~500px`
- 后方生成距玩家 `SCREEN_W - 100~500px`
- 队列满8个时丢弃

### 生成队列消费

`Level_PollSpawns(out, max)` 把队列内容拷出去，用 `memmove` 移除已消费的项。设计目的：level 不知道 enemy 模块的 API，所以不直接调 `Enemy_Create`，而是产出数据让 main.c 去创建。

## 流程状态机

```
MENU ──(Level_StartGame)──→ PLAYING ──(Level_Pause)──→ PAUSED
                                     ←──(Level_Resume)──
                                     
PLAYING ──(Level_GameOver)──→ GAMEOVER ──(Level_StartGame)──→ PLAYING
```

`Level_Update()` 只在 `PLAYING && !paused` 时才执行敌人生成。
