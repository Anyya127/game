# Combat 模块 — 代码架构说明

## 文件

- `combat_api.h` — 对外接口
- `combat.c` — 实现

## 职责

两件事：**管子弹** + **做碰撞**。碰撞检测结果写成"事件"存队列，不直接改血量——由 main.c 消费事件来扣血。

## 数据结构

```c
BulletData g_bullets[MAX_BULLETS];   // 30个子弹槽位（玩家10+敌人20）
Event g_events[MAX_EVENTS];          // 本帧战斗事件队列
int g_combo;                         // 连击计数器
```

### 子弹数据流

```
main.c → Bullet_Create() → 存入 g_bullets[]
       → 每帧 Bullet_UpdateAll() → 按速度移动 + 边界销毁
       → Bullet_GetAll() → 拷贝出去给检查和渲染
```

### 事件数据流

```
Combat_Update() → 碰撞命中 → emit(event) → 写入 g_events[]
main.c → Combat_PollEvents() → 取出消费 → Combat_ClearEvents()
```

## 核心流程：Combat_Update()

遍历两种碰撞类型：

### 碰撞1：子弹碰撞

遍历 `bullets[]` → 对每颗活跃子弹：
- **玩家子弹** → 遍历 `enemies[]` → AABB重叠检测 → 命中：`emit(EVT_HIT)` + 销毁子弹
- **敌人子弹** → 检测 vs 玩家矩形 → 命中：`emit(EVT_PLAYER_HURT)` + 销毁子弹（玩家无敌时跳过）

### 碰撞2：接触伤害

遍历 `enemies[]` → 对每个活着的敌人 → 跟玩家做 AABB 检测 → 命中：`emit(EVT_PLAYER_HURT)`

### AABB 碰撞公式

```c
bool rectsOverlap(...) {
    return x1 < x2 + w2 && x1 + w1 > x2     // X轴重叠
        && y1 < y2 + h2 && y1 + h1 > y2;    // Y轴重叠
}
```

两个矩形在任何一轴上不重叠，整体就不碰撞。

## 子弹管理

`Bullet_Create()` 按类型设不同体积：玩家 36×18（横向长方形），敌人 30×30（正方形）。

`Bullet_UpdateAll()` 每帧 `pos += vel`。超出边界（`MAP_TOTAL + 50`, `SCREEN_H + 100`）自动销毁。

`Bullet_Destroy()` 用 `memset` 清零槽位，`g_bulletCount--`。

## 与其他模块的关系

|--------|--------|
| main.c | `Bullet_Create()` 创建子弹, `Bullet_UpdateAll()` 移动, `Combat_Update()` 判定, `Combat_PollEvents()` 取事件 |
| render.c | `Bullet_GetAll()` 拿子弹列表渲染, `Combat_PollEvents()` 消费事件做特效 |
