# Player 模块 — 代码架构说明

## 文件

- `player_api.h` — 对外暴露的接口
- `player.c` — 实现

## 数据结构

全局变量只有一个 `PlayerData g_player`。因为游戏只有1个玩家，不需要数组管理。

```c
typedef struct {
    Vec2 pos, vel;          // 位置和速度
    int hp, maxHp, lives;   // 血量、命数
    bool grounded;          // 是否着地
    bool wantsShoot;        // 本帧想射子弹（设true后main.c消耗）
    float shootCooldown;    // 射速限制，默认0.25秒一发
    float invTimer;         // 无敌剩余时间
    ...
} PlayerData;
```

## 核心流程：Player_Update()

每帧调用一次，顺序执行5个阶段：

### 阶段1：冷却递减
`shootCooldown` 减去 `dt`，到0后下次射击键按下才能触发。`invTimer` 同样递减，到0后无敌标志清除。

### 阶段2：输入处理 → 改速度
- 按左/右 → `vel.x = ±PLAYER_SPEED`，同时设 `facingRight`
- 按跳跃 + 着地 → `vel.y = PLAYER_JUMP`（负值，向上）
- 按射击 + 冷却OK → `wantsShoot = true`（自己不创建子弹，设标志等 main.c 消费）
- 松方向键 → `vel.x = 0`

### 阶段3：物理
- 重力：`vel.y += GRAVITY` 每帧下坠
- 摩擦力：`vel.x *= FRICTION` 松键后减速
- 位置 += 速度

### 阶段4：平台碰撞
遍历所有 platform，检查水平重叠后分两种情况：
- **下落踩平台**：脚底穿过平台顶面 → 贴上去，`grounded = true`
- **上升撞天棚**：头顶撞平台底面 → 贴上去，`vel.y = 0`

没踩到任何平台再检查默认地面线。X方向碰撞暂未实现。

### 阶段5：边界
- X轴限制在 `[0, MAP_TOTAL - w]`
- Y轴超出 `SCREEN_H + 200` → 摔死

## 战斗接口

`Player_TakeDamage()` 被 combat 事件驱动：
- 无敌中忽略
- 扣血后进入1秒无敌（`invincible = true, invTimer = 1.0f`）
- HP归零 → 扣命；命归零 → PLAYER_DEAD；否则满血重生
